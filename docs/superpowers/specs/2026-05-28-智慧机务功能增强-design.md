# 智慧机务功能增强 — 设计文档

**日期：** 2026-05-28  
**状态：** 已批准  
**场景：** 基地维修（Base MRO）  
**目标用户：** 一线机务人员、班组长/工段长、质检/适航人员（全角色覆盖）  
**设备：** 响应式，支持 PC + 平板 + 手机

---

## 1. 背景与目标

### 现状

系统已有 8 个 MRO 业务域（健康管理、AR协作、排故助手、手册管理、机库管理、工具航材、VR培训、电子工卡），但功能偏向信息展示，缺乏贯穿检修全流程的业务闭环。

### 四大痛点

| 痛点 | 描述 |
|------|------|
| 工卡/任务管理混乱 | 不知道谁在做什么、做到哪步，工序状态不透明 |
| 人员排班与资质管控 | 不清楚谁有资质做哪项工作，排班靠经验和手工 |
| 物料/工具等待 | 航材缺件、工具找不到，经常因等料停工 |
| 审批与质量签署慢 | 关键工序需质检签署，流转慢导致整体进度拖延 |

### 目标

以**电子工卡为主轴**，将派工、执行、质检签署、物料申领、人员资质全部串联成一个完整的检修作业闭环，并通过作业看板大屏实现全局可视化。

---

## 2. 整体架构

### 模块关系图

```
人员资质台账
    ↓ 校验资质
维修计划与派工中心
    ↓ 生成工卡任务
工卡执行（增强）
    ↓               ↓
质检签署流      物料申领工作流
    ↓               ↓
        作业看板大屏
     （汇聚所有模块数据）
```

### 核心实体：维修任务包（Task Package）

- 一个任务包对应一架飞机的一次检修（如 C 检）
- 包含 N 张工卡，每张工卡有 N 个工序步骤
- 派工、物料申领、质检签署都挂在工卡/步骤上
- 人员资质在派工时校验，不合格不允许分配

### 工卡生命周期状态机

```
待派工 → 已派工 → 执行中 → 待质检 → 质检通过 → 已关闭
                              ↓
                         不符合项 → 整改中 → 再次质检
```

## 3. 新增模块详细设计

### 3.1 维修计划与派工中心 `/mro/planning/`

**功能：**
- 创建检修任务包（绑定机号、检修类型、计划起止时间）
- 工卡列表管理（从模板库导入或手动创建）
- 按资质自动匹配可用人员，工段长拖拽确认派工
- 工时估算与人力负荷视图（甘特图简版）

**页面：**

| 页面 | 路由 | 主要操作 |
|------|------|---------|
| 任务包列表 | `/mro/planning/tasks` | 创建、搜索、状态筛选 |
| 任务包详情/派工 | `/mro/planning/tasks/:id` | 工卡管理、人员派工、进度查看 |
| 人力负荷视图 | `/mro/planning/workload` | 甘特图、人员工时分布 |

**权限：**
- 创建/编辑任务包：`planning:task:create`、`planning:task:edit`
- 派工操作：`planning:assign:edit`（工段长角色）
- 只读查看：`planning:task:view`

---

### 3.2 工卡执行增强（升级现有 `/mro/workcard/`）

**在现有 WorkcardList 基础上新增：**
- 工序步骤列表，逐步确认（含完成时间戳、执行人签名）
- 扫码签到/签出（二维码模拟）
- 现场照片上传（拍照记录，支持相机调用）
- 异常上报入口（自动关联工卡+步骤，通知工段长）

**新增页面：**

| 页面 | 路由 | 主要操作 |
|------|------|---------|
| 工卡执行详情 | `/mro/workcard/:id/execute` | 步骤确认、签到、照片、异常上报 |
| 异常上报 | `/mro/workcard/:id/report` | 填写异常描述、上传照片、选择严重等级 |

**权限：**
- 执行步骤：`workcard:step:execute`（一线机务）
- 异常上报：`workcard:report:create`

---

### 3.3 质检签署流 `/mro/quality/`

**功能：**
- 待签署工序列表（按质检员权限过滤，仅显示本人负责的）
- 电子签署（输入工号+密码确认，模拟数字签名）
- 不符合项（NCR）创建与跟踪（关联工卡步骤，触发回退）
- 质量记录归档与导出（PDF格式）

**页面：**

| 页面 | 路由 | 主要操作 |
|------|------|---------|
| 待签署列表 | `/mro/quality/pending` | 筛选、批量查看、跳转签署 |
| 签署详情 | `/mro/quality/sign/:id` | 查看工序记录、电子签署、创建NCR |
| 不符合项管理 | `/mro/quality/ncr` | NCR列表、状态跟踪、关闭确认 |

**权限：**
- 签署操作：`quality:sign:create`（质检/适航角色）
- NCR管理：`quality:ncr:edit`
- 只读：`quality:pending:view`

---

### 3.4 人员资质台账 `/mro/personnel/`

**功能：**
- 人员列表（工种、执照类别、有效期、当前状态）
- 执照到期预警（30/60/90天分级提醒，颜色区分）
- 资质与工卡类型映射关系维护（哪种执照可执行哪类工卡）
- 派工时自动校验（无资质人员不可被分配，前端置灰提示）

**页面：**

| 页面 | 路由 | 主要操作 |
|------|------|---------|
| 人员资质列表 | `/mro/personnel/list` | 搜索、筛选工种/执照、导出 |
| 人员详情 | `/mro/personnel/:id` | 执照详情、历史记录、资质映射 |
| 资质预警 | `/mro/personnel/alerts` | 到期预警列表、按紧急程度排序 |

**权限：**
- 管理操作：`personnel:license:edit`（人力资源/质量管理角色）
- 只读：`personnel:list:view`

---

### 3.5 物料申领工作流 `/mro/material-request/`

**功能：**
- 工卡关联物料清单（BOM视图，从工卡自动带入）
- 一键生成申领单，提交仓库审核
- 申领进度跟踪（待审核 → 已出库 → 已领取）
- 缺料预警推送给工段长（看板红色告警）

**页面：**

| 页面 | 路由 | 主要操作 |
|------|------|---------|
| 申领单列表 | `/mro/material-request/list` | 按状态筛选、创建申领单 |
| 申领单详情 | `/mro/material-request/:id` | 物料明细、审核操作、进度时间线 |

**权限：**
- 创建申领：`material:request:create`（一线机务）
- 审核出库：`material:request:approve`（仓库管理员角色）

---

### 3.6 作业看板大屏 `/mro/dashboard/operation`

**功能：**
- 机库全局视图（每个机位的飞机+任务包进度环形图）
- 实时统计：工卡完成率、在岗人数、异常事项数、待签署数
- 物料缺件告警、质检积压告警（红色醒目显示）
- 支持大屏全屏展示（隐藏侧边导航，F11全屏）

**页面：**

| 页面 | 路由 | 说明 |
|------|------|------|
| 作业看板 | `/mro/dashboard/operation` | 大屏展示，自动刷新（30秒间隔） |

**权限：**
- 查看：`dashboard:operation:view`（所有角色均可访问）

## 4. 模块间联动规则

| 触发动作 | 联动效果 |
|---------|---------|
| 派工时选择人员 | 自动查询资质台账，不合格者置灰不可选，提示缺少的资质类型 |
| 工卡步骤标记"需质检" | 自动在质检签署流生成待签署项，通知对应质检员 |
| 质检发现不符合项(NCR) | 关联工卡步骤回退为"待整改"状态，通知执行人 |
| 工卡创建/导入 | 自动生成关联物料清单（BOM），可人工调整 |
| 物料申领缺件 | 推送消息给工段长，看板显示红色告警 |
| 工卡全部步骤完成 | 任务包进度自动更新，看板实时刷新 |
| 执照临近到期（30天内） | 资质预警页面高亮，派工时额外提示 |

## 5. 响应式设计策略

### 断点

| 设备 | 断点 | 主要场景 |
|------|------|---------|
| 手机 | < 768px | 查看本人任务、消息通知、步骤确认 |
| 平板 | 768px – 1199px | 工卡执行、步骤签到、照片上传、质检签署 |
| PC | ≥ 1200px | 派工管理、资质台账、看板大屏、全功能操作 |

### 各模块适配要点

| 模块 | 平板/手机适配要点 |
|------|----------------|
| 工卡执行 | 步骤卡片式布局，大按钮，扫码入口突出，照片上传一键拍照 |
| 质检签署 | 签署确认弹窗适配触屏，工号输入键盘友好 |
| 物料申领 | 申领单列表卡片化，进度状态醒目 |
| 作业看板 | 竖屏列表+进度条，横屏保留机位网格视图 |
| 派工中心 | 平板仅支持查看派工结果，拖拽操作限 PC |
| 人员资质 | 列表简化列，预警卡片突出到期时间 |

### 实现规范

- 使用 Tailwind CSS 响应式前缀（`sm:` `md:` `lg:`）控制布局切换
- Element Plus 组件在平板场景使用 `size="large"` 提升触控友好性
- 表格在平板下折叠为卡片列表（`<el-card>` 替代 `<el-table>`）
- 看板大屏支持全屏模式，隐藏侧边导航栏

## 8. 实施优先级

建议按以下顺序实施（每批可独立上线）：

| 批次 | 模块 | 原因 |
|------|------|------|
| 第一批 | 人员资质台账 + 维修计划与派工中心 | 其他模块的基础数据依赖 |
| 第二批 | 工卡执行增强 + 物料申领工作流 | 一线人员直接使用，价值最快可见 |
| 第三批 | 质检签署流 | 依赖工卡执行流程稳定后介入 |
| 第四批 | 作业看板大屏 | 汇聚前三批数据，最后集成 |
