# 前端实施计划

## 总体策略

将前端开发分为 **5个阶段**，每个阶段完成一个独立的功能模块，使用 Mock.js 模拟后端数据，确保每个阶段都可以独立测试和验证。

**核心特点**：
- ✅ 使用 Mock.js 模拟后端 API
- ✅ 创建完整的演示页面
- ✅ 无需等待后端开发
- ✅ 后端完成后无缝切换

---

## 第一阶段：前端基础框架 + 登录系统（第1-2周）

### 目标
建立完整的前端项目框架和登录认证系统，为后续功能提供基础。

### 交付物
- ✅ Vue.js 3.x 项目完整初始化
- ✅ Tailwind CSS 样式框架集成
- ✅ 登录页面完整代码（UI + 逻辑）
- ✅ 主页面布局框架（顶部导航、左侧菜单、主内容区）
- ✅ 路由配置和路由守卫
- ✅ Pinia 状态管理（认证模块）
- ✅ Axios HTTP 客户端和拦截器
- ✅ Mock.js 认证接口
- ✅ 通用工具函数和常量
- ✅ 项目文档和开发指南

### 具体任务

#### 1. **项目初始化**
   - 使用 Vite 创建 Vue.js 3.x 项目
   - 安装依赖：Vue Router、Pinia、Axios、Tailwind CSS、Element Plus、Mock.js
   - 配置项目结构和别名
   - 配置环境变量（开发、测试、生产）

#### 2. **样式框架**
   - 集成 Tailwind CSS
   - 创建全局样式文件
   - 定义颜色主题和设计系统
   - 创建通用 CSS 类和工具类

#### 3. **登录页面**（完整代码）
   - 登录表单 UI（用户名、密码、记住密码、登录按钮）
   - 表单验证（前端验证规则）
   - 登录逻辑（调用 Mock API）
   - 错误提示和加载状态
   - 响应式设计（支持移动端）
   - 密码显示/隐藏切换

#### 4. **主页面布局**（完整代码）
   - 顶部导航栏（Logo、用户信息、登出按钮）
   - 左侧菜单栏（树形菜单、收缩展开）
   - 主内容区域（面包屑、页面内容）
   - 响应式设计（支持移动端菜单收缩）

#### 5. **路由配置**（完整代码）
   - 路由定义（登录、主页、404等）
   - 路由守卫（未登录重定向、权限检查）
   - 动态路由加载
   - 路由懒加载优化

#### 6. **状态管理**（完整代码）
   - Pinia 认证模块（用户信息、Token、登录状态）
   - 用户信息存储
   - Token 管理（存储、刷新、清除）
   - 登录/登出状态管理

#### 7. **HTTP 客户端**（完整代码）
   - Axios 实例配置
   - 请求拦截器（自动添加 Authorization Header）
   - 响应拦截器（错误处理、Token 过期处理）
   - 统一的错误处理
   - 请求超时配置

#### 8. **Mock 数据**
   - 创建 Mock 认证接口
   - 模拟登录接口
   - 模拟登出接口
   - 模拟获取用户信息接口
   - 配置 Mock 网络延迟

#### 9. **工具函数**（完整代码）
   - Token 存储/读取/清除
   - 权限检查函数
   - 日期格式化
   - 数据验证函数
   - 常用工具函数

#### 10. **通用组件**（完整代码）
   - 加载动画组件
   - 消息提示组件
   - 确认对话框组件
   - 分页组件（基础版）

#### 11. **项目文档**
   - 项目结构说明
   - 开发指南
   - API 调用示例
   - 常见问题解答

## 前端开发指南

### 开发环境

```bash
# 安装依赖
npm install

# 开发模式启动
npm run dev

# 生产构建
npm run build

# 预览生产构建
npm run preview

# 代码检查
npm run lint

# 代码格式化
npm run format
```

### 开发规范

- 使用 ESLint 进行代码检查
- 使用 Prettier 进行代码格式化
- 缩进：2 个空格
- 分号：必须
- 引号：单引号
- 使用 `<script setup>` 语法

### 常见任务

#### 添加新页面
1. 在 pages/ 目录创建新组件
2. 在 router/routes.js 中添加路由
3. 在 Sidebar.vue 中添加菜单项

#### 添加新 Mock 接口
1. 在 mock/modules/ 目录创建新文件
2. 定义 Mock 接口
3. 在 mock/index.js 中导入

#### 添加新状态
1. 在 store/modules/ 目录创建新模块
2. 定义 state、actions、getters
