# 第一阶段 - 前端框架设计文档

## 1. 项目概述

### 1.1 阶段目标
建立完整的 Vue.js 3.x 前端项目框架和登录认证系统，为后续功能提供基础。

### 1.2 技术栈
- **框架**：Vue.js 3.x
- **构建工具**：Vite
- **样式**：Tailwind CSS
- **路由**：Vue Router 4.x
- **状态管理**：Pinia
- **HTTP 客户端**：Axios
- **UI 组件库**：Element Plus
- **包管理**：npm / yarn

### 1.3 交付物
- ✅ 完整的项目初始化配置
- ✅ 登录页面（UI + 逻辑）
- ✅ 主页面布局框架
- ✅ 路由配置和路由守卫
- ✅ Pinia 状态管理
- ✅ Axios HTTP 客户端和拦截器
- ✅ 通用工具函数
- ✅ 通用组件
- ✅ 开发指南和文档

### 2.2 各目录说明

#### pages/
存放页面级别的 Vue 组件，每个页面对应一个路由。

#### components/
存放可复用的通用组件，不与路由直接关联。

#### api/
存放所有 API 调用函数，按功能模块分类。

#### store/
使用 Pinia 进行状态管理，分为多个模块：
- `auth.js`：用户认证相关状态（用户信息、Token、登录状态）
- `app.js`：应用全局状态（主题、语言、侧边栏状态等）
- `menu.js`：菜单数据状态

#### router/
路由配置和路由守卫。

#### utils/
工具函数和辅助函数。

#### styles/
全局样式和 CSS 变量。

#### constants/
应用常量定义。

## 3. 核心模块设计

### 3.1 认证模块（store/modules/auth.js）

#### 状态（State）
```javascript
{
  user: {
    id: null,
    username: '',
    realName: '',
    avatar: '',
    dept: {},
    roles: []
  },
  token: '',
  refreshToken: '',
  isLoggedIn: false,
  loading: false,
  error: null
}
```

#### 操作（Actions）
- `login(username, password)` - 登录
- `logout()` - 登出
- `getCurrentUser()` - 获取当前用户信息
- `setToken(token)` - 设置 Token
- `clearAuth()` - 清除认证信息

#### 计算属性（Getters）
- `isAuthenticated` - 是否已认证
- `userInfo` - 用户信息
- `hasRole(role)` - 是否拥有某个角色

### 3.2 应用模块（store/modules/app.js）

#### 状态（State）
```javascript
{
  sidebarCollapsed: false,
  theme: 'light',
  language: 'zh-CN',
  notifications: []
}
```

#### 操作（Actions）
- `toggleSidebar()` - 切换侧边栏
- `setTheme(theme)` - 设置主题
- `setLanguage(language)` - 设置语言
- `addNotification(notification)` - 添加通知
- `removeNotification(id)` - 移除通知

### 3.3 菜单模块（store/modules/menu.js）

#### 状态（State）
```javascript
{
  menus: [],
  activeMenu: null,
  breadcrumbs: []
}
```

#### 操作（Actions）
- `setMenus(menus)` - 设置菜单
- `setActiveMenu(menu)` - 设置活跃菜单
- `setBreadcrumbs(breadcrumbs)` - 设置面包屑

## 4. 路由设计

### 4.1 路由结构

```javascript
// 路由分为两类：
// 1. 公开路由（不需要认证）
//    - /login - 登录页面
//    - /404 - 404 页面

// 2. 受保护路由（需要认证）
//    - / - 主页面（Layout）
//      - /dashboard - 仪表板
//      - /system - 系统管理（后续添加）
//        - /system/user - 用户管理
//        - /system/dept - 部门管理
//        - /system/role - 角色管理
//        - /system/menu - 菜单管理
```

### 4.2 路由守卫

#### beforeEach 守卫
1. 检查用户是否已认证
2. 如果未认证且访问受保护路由，重定向到登录页
3. 如果已认证且访问登录页，重定向到首页
4. 检查用户权限（后续实现）

#### afterEach 守卫
1. 更新页面标题
2. 滚动到顶部
3. 关闭加载动画

## 5. HTTP 客户端设计

### 5.1 Axios 实例配置

```javascript
// 基础配置
{
  baseURL: process.env.VITE_API_BASE_URL,
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json'
  }
}
```

### 5.2 请求拦截器

1. 自动添加 Authorization Header（Token）
2. 添加请求时间戳（防重放）
3. 请求日志记录

### 5.3 响应拦截器

1. 检查响应状态码
2. 处理 Token 过期（刷新 Token 或重新登录）
3. 统一错误处理
4. 响应日志记录

### 5.4 错误处理

```javascript
// 错误分类
{
  401: '未授权，请重新登录',
  403: '禁止访问',
  404: '请求的资源不存在',
  500: '服务器内部错误',
  timeout: '请求超时'
}
```

### 12.2 开发规范

#### 命名规范
- 文件名：使用 PascalCase（组件）或 camelCase（工具函数）
- 变量名：使用 camelCase
- 常量名：使用 UPPER_SNAKE_CASE
- CSS 类名：使用 kebab-case

#### 代码风格
- 使用 ESLint 进行代码检查
- 使用 Prettier 进行代码格式化
- 缩进：2 个空格
- 分号：必须
- 引号：单引号

#### 组件开发
- 使用 `<script setup>` 语法
- Props 使用 TypeScript 类型定义（可选）
- 事件使用 emits 定义
- 样式使用 scoped

#### 状态管理
- 使用 Pinia 进行状态管理
- 模块化设计
- 避免直接修改状态，使用 actions

#### API 调用
- 所有 API 调用放在 api/ 目录
- 使用 request 工具进行 HTTP 请求
- 统一错误处理

### 12.3 常见任务

#### 添加新页面
1. 在 pages/ 目录创建新组件
2. 在 router/routes.js 中添加路由
3. 在 Sidebar.vue 中添加菜单项

#### 添加新 API
1. 在 api/ 目录创建新文件
2. 定义 API 函数
3. 在需要的地方导入使用

#### 添加新状态
1. 在 store/modules/ 目录创建新模块
2. 定义 state、actions、getters
3. 在 store/index.js 中注册模块

#### 添加新工具函数
1. 在 utils/ 目录创建新文件
2. 定义工具函数
3. 在需要的地方导入使用

## 14. 安全考虑

### 14.1 认证安全
- Token 存储在 localStorage（可考虑使用 sessionStorage）
- Token 过期自动刷新
- 登出时清除 Token

### 14.2 XSS 防护
- 使用 Vue 的自动转义
- 避免使用 v-html
- 对用户输入进行验证

### 14.3 CSRF 防护
- 使用 CSRF Token（后端实现）
- 使用 SameSite Cookie

### 14.4 数据安全
- 敏感数据不存储在 localStorage
- 使用 HTTPS 传输
- 对敏感数据进行加密

### 20.5 常见问题

**Q: 如何添加新页面？**
A: 
1. 在 `src/pages/` 创建新组件
2. 在 `src/router/routes.js` 添加路由
3. 在 `src/components/Sidebar.vue` 添加菜单项

**Q: 如何添加新 Mock 接口？**
A:
1. 在 `src/mock/modules/` 创建新文件
2. 定义 Mock 接口
3. 在 `src/mock/index.js` 导入

**Q: 如何切换到真实 API？**
A:
1. 在 `.env.production` 设置 `VITE_USE_MOCK=false`
2. 更新 API 调用指向真实后端
3. 确保返回数据格式一致
