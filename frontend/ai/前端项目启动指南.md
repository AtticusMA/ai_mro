# 前端项目启动指南

## 1. 前置要求

### 1.1 系统要求
- Node.js 16.0 或更高版本
- npm 7.0 或更高版本（或 yarn）
- Git

### 1.2 检查环境
```bash
# 检查 Node.js 版本
node --version

# 检查 npm 版本
npm --version
```

---

## 2. 项目初始化

### 2.1 创建项目
```bash
# 使用 Vite 创建 Vue 3 项目
npm create vite@latest frontend -- --template vue

# 进入项目目录
cd frontend
```

### 2.2 项目结构
创建完成后，项目结构如下：
```
frontend/
├── src/
│   ├── App.vue
│   ├── main.js
│   └── style.css
├── public/
├── index.html
├── package.json
├── vite.config.js
└── ...
```

---

## 3. 安装依赖

### 3.1 安装基础依赖
```bash
npm install
```

```bash
npm install vue-router@4 pinia axios tailwindcss postcss autoprefixer element-plus mockjs
```


## 4. 配置 Tailwind CSS

### 4.1 初始化 Tailwind CSS
```bash
npx tailwindcss init -p
```

这会生成两个文件：
- `tailwind.config.js`
- `postcss.config.js`

### 4.2 配置 tailwind.config.js
```javascript
export default {
  content: [
    "./index.html",
    "./src/**/*.{vue,js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

### 4.3 配置 postcss.config.js
```javascript
export default {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
}
```

## 5. 配置 Vite

### 5.1 创建 vite.config.js
```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import path from 'path'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
  server: {
    port: 5173,
    open: true,
  },
})
```

## 11. 故障排查

### 11.1 问题：端口已被占用
```bash
# 使用其他端口启动
npm run dev -- --port 5174
```

### 11.2 问题：依赖安装失败
```bash
# 清除 npm 缓存
npm cache clean --force

# 重新安装依赖
rm -rf node_modules package-lock.json
npm install
```

### 11.3 问题：Mock 数据不工作
- 确保在 `src/main.js` 中正确导入了 Mock
- 检查浏览器控制台是否有错误信息
- 确保 Mock 接口路径与 API 调用路径一致

### 11.4 问题：路由不工作
- 检查 `src/router/index.js` 是否正确导入
- 确保 `src/main.js` 中正确使用了 router
- 检查路由定义中的路径是否正确

## 12. 开发工作流

### 12.1 日常开发
```bash
# 1. 启动开发服务器
npm run dev

# 2. 编辑代码文件
# 浏览器会自动热更新

# 3. 打开浏览器开发者工具
# F12 或 Ctrl+Shift+I

# 4. 查看控制台输出和网络请求
```

### 12.2 提交代码
```bash
# 1. 检查代码
npm run lint

# 2. 格式化代码
npm run format

# 3. 提交到 Git
git add .
git commit -m "feat: add feature description"
git push
```
