# 后端实施计划

## 总体策略

将后端开发分为 **5个阶段**，与前端开发交替进行，每个阶段完成一个独立的功能模块，确保前后端协作顺畅。

**核心特点**：
- ✅ 与前端开发交替进行
- ✅ 基于 API 文档驱动开发
- ✅ 使用 Spring Boot 微服务框架
- ✅ 完整的数据库设计
- ✅ 充分的单元测试和集成测试

---

## 第一阶段：后端框架 + 认证接口（第3-4周）

### 目标
建立后端微服务框架和用户认证系统，为前端提供登录接口。

### 交付物
- ✅ Spring Boot 微服务框架搭建
- ✅ 数据库基础表（用户表、部门表）
- ✅ 用户登录接口
- ✅ JWT Token 生成和验证
- ✅ 用户登出接口
- ✅ 获取当前用户信息接口
- ✅ 密码 BCrypt 加密
- ✅ 统一的异常处理和响应格式
- ✅ 单元测试

### 具体任务

#### 1. **项目初始化**
   - 创建 Spring Boot 项目（使用 Spring Boot 2.7.x 或 3.x）
   - 配置 Maven 或 Gradle 依赖
   - 配置项目结构和包名
   - 配置 application.yml 环境变量

#### 2. **数据库设计**
   - 创建 sys_user 表（用户表）
   - 创建 sys_dept 表（部门表）
   - 添加必要的索引
   - 初始化超管用户

#### 3. **认证接口开发**
   - 实现用户登录接口（POST /api/auth/login）
   - 实现 JWT Token 生成
   - 实现用户登出接口（POST /api/auth/logout）
   - 实现获取当前用户信息接口（GET /api/auth/user-info）
   - 实现 Token 刷新接口（POST /api/auth/refresh-token）

#### 4. **安全配置**
   - 配置 Spring Security
   - 实现 JWT Token 验证
   - 实现密码 BCrypt 加密
   - 配置 CORS

#### 5. **统一处理**
   - 统一的异常处理
   - 统一的响应格式
   - 统一的错误码定义
   - 请求日志记录

#### 6. **单元测试**
   - 编写认证接口的单元测试
   - 编写 JWT Token 的单元测试
   - 编写密码加密的单元测试

### API 接口定义

#### 登录接口
```
POST /api/auth/login
请求体：
{
  "username": "admin",
  "password": "admin123"
}

响应：
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "...",
    "user": {
      "id": 1,
      "username": "admin",
      "realName": "管理员",
      "avatar": "...",
      "dept": { "id": 1, "name": "A集团" },
      "roles": ["admin"]
    }
  }
}
```

#### 获取用户信息接口
```
GET /api/auth/user-info
请求头：Authorization: Bearer {token}

响应：
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "id": 1,
    "username": "admin",
    "realName": "管理员",
    "avatar": "...",
    "dept": { "id": 1, "name": "A集团" },
    "roles": ["admin"],
    "permissions": ["*:*:*"]
  }
}
```

## 后端开发指南

### 开发环境

```bash
# 构建项目
mvn clean install

# 运行项目
mvn spring-boot:run

# 运行测试
mvn test

# 代码检查
mvn checkstyle:check

# 生成 Javadoc
mvn javadoc:javadoc
```

### 开发规范

- 使用 Checkstyle 进行代码检查
- 使用 Spotbugs 进行 Bug 检查
- 缩进：4 个空格
- 命名规范：驼峰命名法
- 注释：使用 Javadoc 注释

### 常见任务

#### 添加新接口
1. 在 controller 目录创建新的 Controller 类
2. 在 service 目录创建新的 Service 类
3. 在 mapper 目录创建新的 Mapper 接口
4. 在 entity 目录创建新的实体类
5. 编写单元测试

#### 添加新数据库表
1. 在 entity 目录创建新的实体类
2. 在 mapper 目录创建新的 Mapper 接口
3. 编写建表 SQL
4. 编写初始化数据 SQL
