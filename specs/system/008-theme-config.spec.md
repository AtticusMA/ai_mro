# Spec: 前端主题配置

## 1. 背景与目标

系统面向不同行业（航天、制造业、医疗、教育、金融、电力、互联网）客户交付，
需要通过系统级配置快速切换前端品牌风格，以贴合目标行业的视觉规范和用户习惯。
管理员在系统配置页面选择预设行业主题后，所有用户（含登录页）立即看到统一的品牌风格，无需发版。

对齐 Charter 目标：完善的企业内部管理后台底座，为后续 AI 能力接入提供一致的品牌体验。

## 2. 范围

### In Scope

- 7 种预设行业主题：互联网、金融、医疗、教育、制造业、电力、航天。
- 系统级主题配置页面：管理员选择主题后即时生效（无需刷新页面）。
- 主题内容：主色调（primary_color）、侧边栏背景色（sidebar_bg）、侧边栏文字色（sidebar_text_color）。
- 主题主色同步覆盖 Element Plus CSS 变量 `--el-color-primary`（及其衍生变量），通过动态注入 `:root` CSS 变量实现。
- 登录页也应用当前主题配色（GET 接口公开，无需 JWT）。
- 用户登录后自动加载当前激活主题；系统未配置时默认使用"互联网"主题。
- 主题配置持久化存储到数据库。

### Out of Scope

- 用户级主题偏好（所有用户统一使用系统级配置，不支持个人覆盖）。
- 自定义颜色调色盘（仅预设行业主题，不支持任意色值输入）。
- 暗黑模式 Dark Mode（后续按需增量加入）。
- 多套主题并存 / 主题版本管理。
- 系统 Logo 上传（首期不做）。

## 3. 用户故事 / 使用场景

- 作为**系统管理员**，我希望在系统配置页面切换行业主题，以便前端界面风格与客户所在行业匹配。
- 作为**普通用户**，我希望打开登录页和登录后看到符合本企业行业特色的界面风格，以便获得统一的品牌体验。

## 4. 功能需求

| ID | 需求 | 验收标准 |
|----|------|----------|
| FR-1 | 主题配置页面 | Given 管理员进入主题配置页，When 页面加载，Then 展示 7 种预设行业主题卡片，当前激活主题高亮标记 |
| FR-2 | 切换主题即时生效 | Given 管理员选择一种主题并点击保存，When 保存成功，Then 当前会话页面立即应用新主题（无需刷新）：主色、侧边栏等视觉元素随之变化，同时将主色注入 `:root` CSS 变量 `--el-color-primary`（及其 light/dark 衍生变量），Element Plus 组件颜色同步更新 |
| FR-3 | 持久化存储 | Given 管理员保存主题配置，When 任意用户刷新页面或重新登录，Then 仍展示最新保存的主题 |
| FR-4 | 登录页加载主题 | Given 用户访问登录页（尚未登录），When 登录页初始化，Then 前端自动拉取当前主题配置并应用（GET 接口无需 JWT） |
| FR-5 | 登录后加载主题 | Given 用户完成登录进入主界面，When 主界面初始化，Then 前端自动拉取当前主题配置并应用 |
| FR-6 | 默认主题兜底 | Given 系统从未配置主题或主题接口请求失败，When 用户访问系统，Then 使用"互联网"主题作为默认值，不报错不白屏 |

## 5. 非功能需求

- **性能**：主题配置接口 P95 < 200ms；CSS 变量切换无明显闪烁（视觉过渡 < 100ms）。
- **安全**：GET 接口公开（无需 JWT，供登录页使用）；PUT 接口须携带有效 JWT Token 且具备 `system:theme:edit` 权限。
- **可用性**：主题读取失败时降级使用默认"互联网"主题，不影响系统正常使用。
- **存储**：主题配置为单条系统级记录，无分页需求。

| 字段 | 类型 | 必填 | 约束 | 说明 |

## 7. 接口契约

| Method | Path | 权限 | 说明 |
|--------|------|------|------|
| GET | /api/system/theme | 公开 | 查询当前主题配置（含完整颜色值，无需 JWT，供登录页使用） |
| PUT | /api/system/theme | system:theme:edit | 更新主题配置 |

### GET /api/system/theme 响应示例

```json
{
  "code": 200,
  "data": {
    "themeCode": "internet",
    "primaryColor": "#667eea",
    "sidebarBg": "#1d1f2b",
    "sidebarTextColor": "#c8c9cc"
  }
}
```

### PUT /api/system/theme 请求体

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| theme_code | String | 是 | 主题标识，枚举值见第 6 节预设表 |

## 8. 权限边界

- 菜单/按钮权限标识：
  - `system:theme:edit` —— 修改系统主题配置
- 数据权限：不涉及（主题为全局配置，无部门过滤）。
- GET 接口**公开**，无需登录即可访问（用于登录页主题加载）。
- PUT 接口须 JWT + `system:theme:edit` 权限。

## 9. 验收标准

- [ ] 7 种预设主题均可正确切换，颜色值与第 6 节枚举一致。
- [ ] 管理员保存主题后当前会话立即生效（无需刷新），Element Plus 组件颜色随之变化。
- [ ] 主题主色正确注入 `:root` CSS 变量 `--el-color-primary`，按钮、链接等组件颜色同步更新。
- [ ] 登录页打开时自动应用当前主题（无需 JWT）。
- [ ] 主题配置持久化：刷新或重新登录后主题不丢失。
- [ ] 无 `system:theme:edit` 权限的用户调用 PUT 接口返回 403。
- [ ] 主题接口失败时前端降级使用"互联网"主题，不报错不白屏。
- [ ] 前端 Mock 数据与接口契约第 7 节保持一致。
