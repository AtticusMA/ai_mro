# 项目宪章

## 1. 愿景

构建一套面向企业内部的 **AI 工作辅助系统**，以完善的权限/数据权限管理为底座，支撑后续 AI 能力的快速接入与扩展。

## 2. 范围

### In Scope（首期）

- 管理后台：登录、部门、用户、角色、菜单/权限、字典 6 大基础模块。
- 五种数据权限（全部 / 本部门 / 本部门及子部门 / 本人 / 自定义部门）。
- RBAC + JWT 认证。
- 前端先行 + Mock 开发，后端补齐。

### Out of Scope（首期不做）

- AI 业务能力（后续按 Spec 增量加入）。
- SSO 登录（仅用户名/密码）。
- 多租户、国际化（保留扩展点）。

## 3. 非目标

- 不替代既有 OA / HR 系统。
- 不做 C 端用户能力。

## 4. 关键约束

| 类别 | 约束 |
|------|------|
| 技术栈 | 前端 Vue 3 + Element Plus + Tailwind；后端 Spring Boot + Nacos + Dubbo + Redis + MySQL 8 |
| 安全 | 密码 BCrypt（不允许 MD5）；JWT；动态 SQL 数据权限；XSS / SQL 注入 / CSRF 防护 |
| 性能 | 接口 P95 < 1s；并发 ≥ 1000 |
| 可用性 | 系统可用性 ≥ 99.9% |
| 文档 | 所有需求/设计/决策必须落入 `specs/` |

## 5. 关键里程碑

| 阶段 | 目标 |
|------|------|
| M1 | 前端骨架 + Mock + 6 大模块 UI 跑通 |
| M2 | 后端认证 + 用户/部门 + JWT |
| M3 | 角色/菜单/字典 + 数据权限拦截器 |
| M4 | 前后端联调 + 安全/性能验收 |

## 6. 角色与责任

| 角色 | 责任 |
|------|------|
| 产品 | Charter / Spec |
| 架构 | Plan / ADR |
| 开发 | Tasks / Code |
| QA | 验收标准执行 |

## 7. 变更治理

- Charter 变更须经产品 + 架构双方签字，bump major 版本。
- 任何与本宪章冲突的 Spec/Plan 必须先变更 Charter，否则不得 approved。
